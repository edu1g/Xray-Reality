# 安全策略

## 受支持的版本

下表列出了目前仍在接收安全更新的项目版本。

| 版本号  | 是否支持           |
| ------- | ------------------ |
| 1.3.5   | :white_check_mark: |
| < 1.3.5 | :x:                |

## 报告漏洞

请在您的报告中尽可能提供以下信息：
* 漏洞类型（例如：跨站脚本攻击、SQL 注入等）
* 复现该问题所需的步骤
* 预期行为以及实际发生的情况
* 任何概念验证（PoC）代码或截图

所有的安全漏洞都将得到迅速处理。我们将尽量在 48 小时内确认收到您的漏洞报告，并定期向您同步修复进展。

# Security Policy

## Supported Versions

The following table outlines which versions of the project are currently receiving security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.3.5   | :white_check_mark: |
| < 1.3.5 | :x:                |

## Reporting a Vulnerability

Please include as much of the following information as possible in your report:
* Type of vulnerability (e.g., Cross-Site Scripting, SQL Injection, etc.)
* The steps required to reproduce the issue
* The expected behavior and what actually happens
* Any proof-of-concept (PoC) code or screenshots

All security vulnerabilities will be promptly addressed. We will try to acknowledge receipt of your vulnerability report within 48 hours and provide regular updates on the resolution process.
